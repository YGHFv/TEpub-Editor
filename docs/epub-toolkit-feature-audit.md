## 对照结论

### 本项目原本已有

| 外部功能 | TEpub Editor 对应能力 |
| --- | --- |
| TXT 转 EPUB | TXT 编辑器与 EPUB 制作流程 |
| 编辑元数据、查看 OPF | 书库元数据编辑、Web/桌面 EPUB 编辑器 |
| 替换封面 | 书库封面写回与 EPUB 编辑器资源替换 |
| 重新格式化 | EPUB 重构 |
| WebP 转 JPEG/PNG | 图片转换 |
| 名称混淆加密/解密 | 文件加密/解密 |
| 字体加密 | 字体加密/解密 |

### 本次双端迁移

以下能力使用共享 TypeScript 处理内核，安装版与 Web 版进入相同页面并执行相同代码：

| 新功能 | 实现摘要 |
| --- | --- |
| EPUB 转 TXT | 按 spine 阅读顺序提取正文并输出 UTF-8 TXT |
| EPUB 2.0 / 3.0 互转 | 更新 OPF 版本并补齐 NCX/NAV 导航 |
| EPUB 简繁转换 | 使用 OpenCC 转换正文、导航和元数据文本节点 |
| EPUB 广告清理 | 支持逐行正则，限制在短段落节点内删除，降低误删风险 |
| EPUB 拼音标注 | 使用标准 XHTML `ruby` / `rt` 元素和带声调拼音 |
| 批注与脚注转换 | 标准脚注弹窗数据增强；属性型弹窗批注转 EPUB3 `noteref` / `footnote` |
| 图片压缩 | JPEG/WebP 质量压缩、PNG/大图缩放，仅在输出更小时写回 |
| 图片格式双向转换 | 原 WebP→PNG/JPEG 扩展为 PNG/JPEG↔WebP |
| 图片隐形水印 | 自有 `TEPUBWM2` LSB 格式，包含长度和 FNV-1a 校验，可写入和扫描 |
| 合并 EPUB | 为每本来源书分配独立资源目录，重写相对引用并重建 OPF/NAV/NCX |
| 拆分 EPUB | 预览 spine 条目，按章节数生成可独立加载的 EPUB |
| 字体子集化 | 收集正文实际 Unicode 码点，裁剪 TTF/OTF/WOFF/WOFF2，结果不变小时不写回 |
| Send to Kindle 网页 | 双端打开 Amazon 官方页面，不在应用内收集 Amazon 凭据 |

## 暂不迁移

| 外部功能 | 原因与后续条件 |
| --- | --- |
| SMTP Kindle 邮件推送 | 外部项目自身标记为未完整实机验证；Web 端还会引入邮箱凭据存储和服务端代发风险。当前采用官方 Send to Kindle 页面。 |
| 下载 EPUB 网络图片 | 静态 Web 受跨域限制；若增加服务端/原生代理，必须先实现 DNS 解析后的私网地址拦截、重定向复检、大小/类型限制，避免 EPUB 触发 SSRF。 |
| 阅微/得到/掌阅到多看专用脚注 | 当前先提供标准 EPUB3 与通用属性型弹窗转换。平台私有格式需要真实样本和回归夹具，不能只按正则猜测。 |
| ZIP 密码处理 | 该目录存在于外部源码，但未进入其当前路由和公开功能目录，不作为当前版本对照功能。 |

## 验证覆盖

`src/lib/webEpubAdvanced.test.ts` 使用运行时生成的真实 EPUB ZIP 覆盖：

- spine 顺序 TXT 提取；
- EPUB 3→2 OPF/NCX 转换；
- 简体转繁体；
- 广告段落清理；
- ruby 拼音标注；
- 标准脚注弹窗增强；
- 多书合并后重新加载；
- 拆分产物逐本重新加载；
- 图片水印载荷写入、读取和校验失败；
- Windows 存在系统中文字体时的真实 TTF 子集化。
- 合并 EPUB 的 ZIP 第一项为未压缩 `mimetype`，满足 OCF 容器顺序要求；拆分流程复用同一构建器。
- 字体子集扫描保留通过十六进制和十进制 XHTML 数字实体引用的 Unicode 码点。

Canvas 图片编码仍需在浏览器运行时执行；生产 Web/安装版前端构建和各工具路由用于验证其打包与可达性。

## 2026-07-16 复审补充

- 复审确认合并/拆分初版在复制资源后才插入 `mimetype`，部分严格阅读器会拒绝这种 ZIP；现已调整为创建归档时首先写入并以 STORE 方式保存。
- 复审确认字体子集初版跳过了 `&#x...;` 与 `&#...;` 数字实体；现已解析有效 Unicode 标量并加入保留集合。
- 繁体转简体与简体转繁体现在使用对应的台湾繁体配置成对转换，不再混用香港繁体来源配置。
- 标准 EPUB3、资源重写、合并/拆分重新加载及字体实体覆盖均有自动化回归；Canvas 图片重编码仍依赖真实浏览器能力。
- 阅微、得到、掌阅等平台私有脚注仍缺少合法可用于回归的真实样本，继续维持“不按猜测正则迁移”的边界。
